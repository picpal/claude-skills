# 직무 적합성 분석 가이드

## JD가 있을 때: 요구사항 매칭 분석

JD(채용공고)가 제공되면 체계적으로 매칭 분석을 수행한다.

### 매칭 프레임워크

JD에서 다음을 추출한다:

1. **필수 자격요건** — "~필수", "~경험자", "~자격 보유자"
2. **우대 사항** — "~우대", "~있으면 좋음", "~경험자 우대"
3. **핵심 키워드** — 반복적으로 등장하는 기술, 역량, 도메인 용어
4. **문화 키워드** — 팀 구조, 업무 방식, 회사가 중시하는 가치

### 커버리지 매트릭스

각 요구사항에 대해 이력서가 얼마나 커버하는지 판단한다:

| 커버리지 수준 | 설명 |
|--------------|------|
| 직접 매치 | 이력서에 해당 경험/기술이 명시적으로 기술됨 |
| 간접 매치 | 유사한 경험이 있으나 직접적 언급이 없음 |
| 미커버 | 이력서에 관련 내용이 전혀 없음 |

### 첨삭 방향

- **직접 매치**: JD 키워드와 동일한 용어를 사용하도록 표현 조정
- **간접 매치**: 관련 경험을 JD 맥락에 맞게 재구성하여 연결고리를 명시
- **미커버**: 보완 가능한 경험이 있으면 추가 서술 제안, 없으면 솔직히 안내

## JD가 없을 때: 일반 직무 기준 분석

JD가 없으면 직무명과 업계 일반 기준으로 분석한다.

### 직무별 핵심 역량 체크리스트

#### 소프트웨어 엔지니어 / 개발자
- 주력 기술 스택이 명확한가
- 코드 품질/테스트/CI-CD 경험이 드러나는가
- 문제 해결 사례가 있는가
- 협업 도구/프로세스 경험 (Git, 코드리뷰, 애자일 등)

#### 데이터 분석가 / 사이언티스트
- 분석 도구/언어 숙련도 (SQL, Python, R 등)
- 비즈니스 임팩트가 있는 분석 사례
- 시각화/커뮤니케이션 능력
- 통계/ML 기법 활용 경험

#### 기획자 / PM / PO
- 제품 런칭/개선 성과
- 이해관계자 관리 경험
- 데이터 기반 의사결정 사례
- 와이어프레임/PRD 작성 경험

#### 디자이너 (UX/UI)
- 포트폴리오 링크 포함 여부
- 사용자 리서치/테스트 경험
- 디자인 시스템 구축/활용
- 비즈니스 메트릭 개선 기여

#### 마케터
- 캠페인 기획/실행 성과 (ROAS, CPA 등)
- 채널별 전문성 (퍼포먼스, 콘텐츠, CRM 등)
- 데이터 분석 역량
- A/B 테스트/실험 경험

### 키워드 전략

이력서에 해당 직무의 업계 표준 키워드가 포함되어 있는지 확인한다.
ATS(지원자 추적 시스템)를 통과하려면 JD에서 사용하는 용어와 동일한 키워드를 사용하는 것이 중요하다.

예시:
- "코딩"보다 "소프트웨어 개발", "프로그래밍"
- "그림 그리기"보다 "UI 디자인", "비주얼 디자인"
- "홍보"보다 "마케팅 커뮤니케이션", "PR"
