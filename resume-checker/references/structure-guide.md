# 이력서 구조 가이드

## 필수 섹션

모든 이력서에 포함되어야 하는 기본 섹션이다.

### 1. 인적사항 / 헤더

- 이름, 연락처(이메일, 전화번호)
- 포트폴리오/GitHub/LinkedIn 링크 (해당 직무에 관련 있을 경우)
- 주소는 시/구 수준까지만 (상세 주소 불필요)
- 사진은 요구하지 않는 한 불필요

**흔한 실수:**
- SNS 계정을 무분별하게 나열 (직무 관련 링크만 포함)
- 생년월일, 성별, 병역 등 불필요한 개인정보 과다 기재

### 2. 경력사항

- 최신 경력부터 역순으로 정렬 (역순 연대기)
- 각 경력: 회사명, 직책, 근무 기간, 주요 업무/성과
- 근무 기간은 "YYYY.MM ~ YYYY.MM" 형식 통일
- 각 항목에 3~5개의 불릿 포인트로 핵심 업무/성과 기술

**흔한 실수:**
- 업무 내용만 나열하고 성과가 없음
- 기간 표기 불일치 (어떤 곳은 년/월, 어떤 곳은 년만)
- 너무 오래된 경력까지 상세하게 기술 (10년 이상 경력은 최근 5~7년 중심)

### 3. 학력

- 최종 학력부터 역순 정렬
- 학교명, 전공, 졸업 연도
- 신입의 경우 GPA 포함 가능 (3.5/4.5 이상일 때)
- 경력 5년 이상이면 간략하게 (학교, 전공, 졸업연도만)

### 4. 기술 스택 / 스킬

- 직무와 직접 관련된 기술만 나열
- 숙련도 표시가 있으면 좋음 (상/중/하 또는 연차)
- 카테고리별 그룹핑 (언어, 프레임워크, 도구 등)

## 선택 섹션

직무나 경력 수준에 따라 포함 여부를 판단한다.

### 자기소개 / Summary

- 2~3문장으로 핵심 역량과 지원 동기를 요약
- 경력직에게 특히 효과적
- "성실하고 책임감 있는" 같은 추상적 문구 지양
- 구체적인 수치와 성과 기반 요약이 이상적

### 프로젝트

- 개인/사이드 프로젝트, 오픈소스 기여
- 각 프로젝트: 이름, 기간, 역할, 기술 스택, 성과
- 신입이나 직무 전환자에게 특히 중요

### 자격증 / 수상

- 직무 관련 자격증만 포함
- 취득일 명시
- 운전면허 등 무관한 자격증은 제외 (직무상 필요한 경우 예외)

### 교육 / 활동

- 부트캠프, 직무 교육, 컨퍼런스 발표 등
- 직무 관련성이 높은 것만 선별

## 섹션 순서 가이드

경력 수준에 따라 권장 순서가 다르다.

**신입/주니어 (경력 0~3년):**
1. 인적사항
2. 자기소개
3. 기술 스택
4. 프로젝트
5. 학력
6. 자격증/수상

**경력직 (3년 이상):**
1. 인적사항
2. 자기소개 (선택)
3. 경력사항
4. 기술 스택
5. 프로젝트 (선택)
6. 학력
7. 자격증/수상 (선택)

## 분량 가이드

| 경력 | 권장 분량 |
|------|-----------|
| 신입 | 1페이지 |
| 1~5년 | 1~2페이지 |
| 5~10년 | 2페이지 |
| 10년+ | 2~3페이지 (최근 중심) |
