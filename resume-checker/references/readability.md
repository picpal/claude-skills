# 가독성 및 포맷 가이드

## 불릿 포인트

### 일관성 규칙

- 한 섹션 내에서 불릿 스타일 통일 (-, *, • 중 하나)
- 들여쓰기 레벨 통일 (최대 2단계까지)
- 각 불릿의 길이를 비슷하게 유지 (한 줄이 너무 길면 두 줄로 분리)
- 불릿 시작을 동사/명사로 통일

**나쁜 예 (불일치):**
```
- API를 개발했습니다
* 데이터베이스를 설계함
• 서버 모니터링
  - Prometheus 사용
```

**좋은 예 (일관적):**
```
- 주문 처리 API 설계/구현 (Spring Boot, JPA)
- 데이터베이스 스키마 설계 및 쿼리 최적화
- Prometheus 기반 서버 모니터링 시스템 구축
```

## 날짜/기간 표기

### 통일 규칙

한 가지 형식을 선택하여 전체 이력서에 일관 적용한다.

| 형식 | 예시 |
|------|------|
| YYYY.MM | 2022.03 ~ 2024.06 |
| YYYY년 MM월 | 2022년 3월 ~ 2024년 6월 |
| YYYY/MM | 2022/03 ~ 2024/06 |

**주의:**
- "2022.3"과 "2022.03" 혼용하지 않기
- "~", "–", "-" 중 하나로 통일
- 재직 중이면 "~ 현재" 또는 "~ Present"

### 기간 계산

- 근무 기간을 괄호로 표시하면 가독성이 좋다
- 예: "2022.03 ~ 2024.06 (2년 4개월)"

## 길이 가이드

### 경력별 적정 분량

| 경력 | 페이지 | 주의사항 |
|------|--------|----------|
| 신입 | 1장 | 관련 없는 아르바이트/동아리 과다 기재 주의 |
| 1~3년 | 1~1.5장 | 프로젝트를 선별적으로 |
| 3~7년 | 1.5~2장 | 핵심 경력 중심 |
| 7~15년 | 2장 | 최근 5~7년 상세, 이전은 요약 |
| 15년+ | 2~3장 | 핵심 경력만, 나머지는 1줄 요약 |

### 불릿 포인트 수

- 주요 경력: 항목당 3~5개
- 부수적 경력: 항목당 2~3개
- 프로젝트: 항목당 2~4개

## 시각적 구조

### 정보 밀도

- 적절한 줄 간격과 섹션 간 여백
- 섹션 구분이 시각적으로 명확한가 (구분선, 볼드 제목 등)
- 한 불릿에 너무 많은 정보를 넣지 않기 (2줄 이내)

### 키워드 가시성

- 기술 스택, 도구명은 시각적으로 구분 (볼드, 괄호 등)
- 핵심 성과 수치는 눈에 잘 띄게
- 예: "응답 시간 **83% 단축** (2.3초 → 0.4초)"

### 한글 이력서 특유 주의사항

- 한영 혼용 시 띄어쓰기 일관성
- 영문 기술명은 원어 그대로 (예: React, Spring Boot — "리액트", "스프링부트"로 바꾸지 않기)
- 단, 일반 명사화된 외래어는 한글 표기 (예: 서버, 데이터베이스, 프레임워크)
