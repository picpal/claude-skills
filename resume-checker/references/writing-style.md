# 문체 및 표현 가이드

## 어미 일관성

이력서 전체에서 하나의 어미 스타일을 일관되게 사용해야 한다.

| 스타일 | 예시 | 적합한 상황 |
|--------|------|-------------|
| 명사형 종결 | "매출 30% 증가 달성" | 간결한 불릿 포인트 |
| ~함/~했음 | "매출 30% 증가를 달성했음" | 서술형 경력 기술 |
| ~하다/~했다 | "매출 30% 증가를 달성했다" | 자기소개서 톤 |

**주의:** 하나의 이력서 안에서 "~했습니다"와 "~했음"을 혼용하면 안 된다.

## 흔한 오탈자/맞춤법 오류

### 자주 틀리는 표현

| 틀린 표현 | 올바른 표현 | 설명 |
|-----------|-------------|------|
| 되요 | 돼요 | "되어요"의 축약 |
| 됬다 | 됐다 | "되었다"의 축약 |
| 웹 어플리케이션 | 웹 애플리케이션 | 외래어 표기법 |
| 데이타 | 데이터 | 외래어 표기법 |
| 메세지 | 메시지 | 외래어 표기법 |
| 컨텐츠 | 콘텐츠 | 외래어 표기법 |
| 라이센스 | 라이선스 | 외래어 표기법 |
| 악세사리 | 액세서리 | 외래어 표기법 |
| 가르키다 | 가리키다 | |
| ~로서 (수단) | ~로써 | "~의 자격으로"는 로서, "~을 이용하여"는 로써 |

### 띄어쓰기 주의

| 틀린 표현 | 올바른 표현 |
|-----------|-------------|
| 할수있다 | 할 수 있다 |
| 할것이다 | 할 것이다 |
| 그동안 | 그동안 (붙여씀) |
| 웹개발 | 웹 개발 |
| 3년간 | 3년간 (붙여씀) |

## 표현 개선 가이드

### 추상적 → 구체적

| 추상적 (피해야 할 표현) | 구체적 (권장 표현) |
|------------------------|-------------------|
| 다양한 경험을 보유 | 3개 프로젝트에서 프론트엔드 리드 경험 |
| 열심히 노력함 | 주 60시간 이상 투자하여 3개월 내 출시 달성 |
| 좋은 성과를 냄 | 전년 대비 매출 23% 증가 기여 |
| 여러 기술을 활용 | React, TypeScript, GraphQL 기반 SPA 구축 |
| 팀원들과 협업 | 5인 크로스펑셔널 팀에서 프로젝트 매니저 역할 수행 |
| 성능을 개선함 | API 응답 시간 2.3초 → 0.4초로 83% 단축 |
| 안정적인 서비스 운영 | 연간 가용률 99.95% 달성, 장애 대응 MTTR 15분 이내 |

### 수동태/피동 → 능동

| 피동 | 능동 |
|------|------|
| ~에 참여하게 되었음 | ~에 참여하여 |
| ~를 담당하게 됨 | ~를 주도/리드함 |
| ~가 진행되었음 | ~를 진행함 |

### 불필요한 수식어

다음 표현은 이력서에서 정보를 전달하지 않으므로 삭제하거나 구체적 내용으로 대체한다:

- "성실하고 책임감 있는"
- "커뮤니케이션 능력이 뛰어난"
- "빠른 학습 능력을 보유한"
- "열정적인"
- "꼼꼼한"

이런 성향은 경험과 성과로 보여줘야 한다. "커뮤니케이션 능력이 뛰어남" 대신 "주간 스프린트 회고 리드, 비개발 이해관계자 대상 기술 브리핑 월 2회 진행"처럼 행동으로 입증하는 것이 효과적이다.
