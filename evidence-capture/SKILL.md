# Evidence Capture (증적 캡처)

감사/인증을 위한 증적 자료를 자동으로 캡처하여 PNG 파일로 저장하는 스킬이다.

## 캡처 대상

| 유형 | 설명 | 방법 |
|------|------|------|
| **소스코드** | 보안 로직, 키 설정, 세션 설정 등 코드 특정 부분 | pygments로 구문 강조 후 PNG 렌더링 |
| **웹페이지** | 로그인 화면, 대시보드, 관리 콘솔 등 | Playwright로 스크린샷 |
| **터미널 출력** | 명령어 실행 결과 (버전 확인, 설정 조회 등) | 명령 실행 후 HTML→PNG 변환 |
| **로그 출력** | 접근 로그, 감사 로그, 에러 로그 등 | 로그 읽기 후 HTML→PNG 변환 |

## 워크플로우

### 1단계: 요청 분석

사용자의 증적 요구사항을 파악한다. 요청에는 보통 다음이 포함된다:
- 캡처할 대상 (파일 경로, URL, 명령어 등)
- 증적 항목명 또는 번호 (예: "A-1. 비밀번호 암호화 처리")
- 캡처 범위 (특정 함수, 특정 설정 값 등)

요청이 모호하면 구체적으로 무엇을 캡처할지 확인한다.

### 2단계: 출력 디렉토리 준비

```
evidence_output/
├── 01_비밀번호_암호화/
│   ├── capture_001.png
│   └── capture_002.png
├── 02_세션_타임아웃/
│   └── capture_001.png
└── evidence_index.md    ← 증적 목록 인덱스
```

- 출력 디렉토리 기본 경로: 현재 작업 디렉토리 하위 `evidence_output/`
- 사용자가 경로를 지정하면 해당 경로 사용
- 증적 항목별로 하위 디렉토리를 만들어 정리
- 디렉토리명은 번호_항목명 형식으로, 한글/영문 모두 가능 (공백은 언더스코어로 치환)

### 3단계: 캡처 실행

각 유형별 캡처 방식:

#### 소스코드 캡처

`scripts/capture_code.py` 스크립트를 사용한다.

```bash
python3 <skill-path>/scripts/capture_code.py \
  --file <소스파일경로> \
  --lines <시작줄>-<끝줄> \
  --output <출력PNG경로> \
  --title "증적 항목명"
```

- 파일 경로와 줄 범위를 지정하여 해당 부분만 캡처
- 줄 번호 표시, 구문 강조(syntax highlighting) 적용
- 상단에 파일 경로와 증적 항목명을 헤더로 표시
- 줄 범위를 지정하지 않으면 파일 전체를 캡처 (단, 100줄 초과 시 사용자에게 범위 확인)

#### 웹페이지 캡처

Playwright CLI를 사용한다.

```bash
npx playwright screenshot --full-page <URL> <출력PNG경로>
```

- 전체 페이지 캡처가 기본
- 로그인이 필요한 페이지는 사용자에게 안내 (직접 로그인 후 쿠키/세션 제공 필요)
- 특정 영역만 필요하면 `--clip` 옵션 사용 가능

#### 터미널 명령 실행 결과 캡처

`scripts/capture_terminal.py` 스크립트를 사용한다.

```bash
python3 <skill-path>/scripts/capture_terminal.py \
  --command "<실행할 명령어>" \
  --output <출력PNG경로> \
  --title "증적 항목명"
```

- 명령어를 실행하고 stdout/stderr를 캡처
- 터미널 스타일(검정 배경, 녹색/흰색 텍스트)로 렌더링
- 상단에 실행한 명령어와 증적 항목명을 헤더로 표시

#### 로그 파일 캡처

`scripts/capture_terminal.py`를 재활용하여 로그 내용을 캡처한다.

```bash
python3 <skill-path>/scripts/capture_terminal.py \
  --file <로그파일경로> \
  --lines <시작줄>-<끝줄> \
  --output <출력PNG경로> \
  --title "증적 항목명"
```

- 로그 파일의 특정 범위를 읽어서 터미널 스타일로 렌더링
- 줄 범위를 지정하지 않으면 마지막 50줄을 캡처

### 4단계: 인덱스 생성

모든 캡처가 완료되면 `evidence_index.md`를 생성한다:

```markdown
# 증적 자료 목록

생성일: 2024-01-15

| 번호 | 증적 항목 | 파일 | 캡처 유형 | 설명 |
|------|-----------|------|-----------|------|
| 1 | 비밀번호 암호화 | 01_비밀번호_암호화/capture_001.png | 소스코드 | auth/password.py:45-62 |
| 2 | 세션 타임아웃 | 02_세션_타임아웃/capture_001.png | 소스코드 | config/session.yml:10-15 |
```

### 5단계: 결과 보고

캡처 완료 후 사용자에게 결과를 보고한다:
- 총 캡처 건수
- 각 증적 항목별 캡처 파일 경로
- 캡처 실패 건이 있으면 사유와 함께 안내

## 주의사항

- 민감 정보(실제 비밀번호, API 키 값 등)가 캡처에 포함되지 않도록 주의한다. 캡처 전에 민감 정보가 노출되는지 확인하고, 노출되면 사용자에게 알린다.
- 웹페이지 캡처 시 인증이 필요한 페이지는 사용자가 직접 접근 가능한 상태여야 한다.
- 캡처 이미지의 해상도는 읽기 편한 수준을 유지한다 (코드: 폰트 크기 14pt 이상, 웹: 1280px 너비 기본).
